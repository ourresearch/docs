Topics are research areas automatically assigned to works. Topics exist in a four-level hierarchy: domain > field > subfield > topic.

## Fields

<ResponseField name="id" type="string">
  The OpenAlex ID for this topic.

  `"https://openalex.org/T11636"`
</ResponseField>

<ResponseField name="display_name" type="string">
  The English-language label of the topic.

  `"Artificial Intelligence in Medicine"`
</ResponseField>

<ResponseField name="description" type="string">
  A description of this topic, generated by AI.

  `"This cluster of papers explores the intersection of artificial intelligence and medicine..."`
</ResponseField>

<ResponseField name="keywords" type="array">
  Keywords representing the content of papers in the topic.

  `["Artificial Intelligence", "Machine Learning", "Healthcare", "Medical Imaging"]`
</ResponseField>

<ResponseField name="domain" type="object">
  The domain (highest level) containing this topic:
  - `id`: Domain ID
  - `display_name`: Domain name

  ```json
  {
    "id": 4,
    "display_name": "Health Sciences"
  }
  ```
</ResponseField>

<ResponseField name="field" type="object">
  The field (second level) containing this topic:
  - `id`: Field ID
  - `display_name`: Field name

  ```json
  {
    "id": 27,
    "display_name": "Medicine"
  }
  ```
</ResponseField>

<ResponseField name="subfield" type="object">
  The subfield (third level) containing this topic:
  - `id`: Subfield ID
  - `display_name`: Subfield name

  ```json
  {
    "id": 2718,
    "display_name": "Health Informatics"
  }
  ```
</ResponseField>

<ResponseField name="works_count" type="integer">
  The number of works tagged with this topic.

  `21737`
</ResponseField>

<ResponseField name="ids" type="object">
  External identifiers:
  - `openalex`: OpenAlex ID
  - `wikipedia`: Wikipedia page URL
</ResponseField>

<ResponseField name="updated_date" type="string">
  When this topic was last updated (ISO 8601 datetime).

  `"2024-02-05T05:00:03.798420"`
</ResponseField>
