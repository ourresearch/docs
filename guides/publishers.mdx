title: Publishers
description: Understanding publisher entities in OpenAlex
---

Publishers are companies and organizations that publish scholarly works. OpenAlex indexes over 10,000 publishers.

```bash
# List all publishers
https://api.openalex.org/publishers

# Get a specific publisher
https://api.openalex.org/publishers/P4310319965
```

## Key properties

| Property | Description |
|----------|-------------|
| `id` | OpenAlex ID (e.g., `P4310319965`) |
| `ror` | ROR identifier |
| `display_name` | Publisher name |
| `country_codes` | Countries of operation |
| `works_count` | Number of works |
| `cited_by_count` | Total citations |
| `sources_count` | Number of sources (journals) |
| `hierarchy_level` | 0 = parent, 1+ = subsidiary |
| `lineage` | Parent publisher(s) |
| `parent_publisher` | Immediate parent |

For a comprehensive field listing, see the [Get a single publisher](/api-reference/publishers/get-a-single-publisher) response specification.

See the [Publisher object](/api-reference/publishers) for the complete schema.

## Publisher hierarchy

Publishers form a hierarchy. For example, "Elsevier" is a parent publisher with subsidiaries like "Cell Press" and "The Lancet".

| Property | Description |
|----------|-------------|
| `hierarchy_level` | 0 = top-level parent |
| `parent_publisher` | Direct parent |
| `lineage` | All ancestors |

```bash
# Get only top-level publishers
https://api.openalex.org/publishers?filter=hierarchy_level:0

# Get subsidiaries of Elsevier
https://api.openalex.org/publishers?filter=parent_publisher:P4310319965
```

## Common filters

| Filter | Example |
|--------|---------|
| By country | `filter=country_codes:US` |
| Top-level only | `filter=hierarchy_level:0` |
| Has ROR | `filter=has_ror:true` |
| High output | `filter=works_count:>100000` |

```bash
# Large publishers in the UK
https://api.openalex.org/publishers?filter=country_codes:GB,works_count:>10000&sort=-works_count
```
